

# Propuesta Comercial ALICIA
## Transformación del Canal de Pedidos WhatsApp para Crepes & Waffles

---

## Resumen Ejecutivo

Esta propuesta presenta la implementación de ALICIA, un asistente de IA conversacional diseñado para reemplazar el chatbot transaccional actual de Crepes & Waffles por WhatsApp. El objetivo es transformar un canal de deflexión de tráfico en un verdadero canal de ventas que represente la experiencia premium de la marca, consolidando múltiples canales fragmentados en una sola experiencia de excelencia.

---

## Diagnóstico: El Problema Actual

### Situación Documentada

El chatbot actual de Crepes & Waffles funciona como un formulario disfrazado de conversación:

| Aspecto | Estado Actual |
|---------|---------------|
| Propósito | Desviar tráfico humano, no vender |
| Inteligencia | Sin contexto ni memoria conversacional |
| Experiencia | Robótica, frustrante |
| Geolocalización | Fallas documentadas (ubicaciones cercanas no detectadas) |
| Resultado | Clientes migran a Rappi/iFood |

### Caso Real Documentado

Un cliente envió su ubicación y recibió "No hay Crepes & Waffles disponibles cerca", cuando había una sucursal literalmente al lado. Este error se repitió en múltiples intentos, evidenciando una falla sistemática en la geolocalización.

### El Problema del Call Center

Crepes & Waffles también opera un call center para recibir pedidos. Aunque ofrece atención humana, este canal presenta riesgos inherentes:

| Aspecto | Riesgo |
|---------|--------|
| Variabilidad emocional | Una operadora de mal humor = cliente perdido |
| Inconsistencia | Cada llamada depende del estado de ánimo del día |
| Escalabilidad | Contratar más personal = más costos fijos |
| Horarios | Limitado a turnos operativos |
| Capacitación | Requiere entrenamiento continuo |

**El dilema actual:** El call center ofrece calidez humana pero con riesgo emocional. El bot actual ofrece consistencia pero con experiencia robótica. Ninguno de los dos representa la marca premium de Crepes & Waffles.

### Canales Fragmentados

Actualmente, los clientes tienen múltiples opciones que compiten entre sí:

| Canal | Problema |
|-------|----------|
| WhatsApp (bot actual) | Fallas de geolocalización, experiencia robótica |
| Call Center | Costoso, riesgo emocional, escalabilidad limitada |
| Página Web | Compleja, baja adopción, frialdad transaccional |
| Rappi / iFood | Comisiones 15-30%, pérdida de datos del cliente |

Esta fragmentación genera confusión, inconsistencia de marca y pérdida de oportunidades de venta directa.

### Impacto Financiero del Problema Actual

Escenario conservador para 200 sucursales:

| Métrica | Cálculo |
|---------|---------|
| Intentos de pedido fallidos por sucursal/día | 5-10 |
| Valor promedio pedido domicilio | $45.000 COP |
| Ventas perdidas diarias (200 sucursales) | $45M - $90M COP |
| Ventas perdidas mensuales | $1.350M - $2.700M COP |
| Ventas perdidas anuales | $16.200M - $32.400M COP |

Además, cada cliente frustrado representa:
- Migración a plataformas terceras (Rappi: 15-30% de comisión)
- Deterioro de la percepción de marca premium
- Pérdida de datos de cliente directo

### Por Qué WhatsApp

WhatsApp es el canal natural para Crepes & Waffles:

| Razón | Alineación con la Marca |
|-------|------------------------|
| **Intimidad** | WhatsApp es personal, como la experiencia en el restaurante |
| **Cercanía** | La gente ya tiene WhatsApp abierto todo el día |
| **Tradición** | Es el canal más directo, sin intermediarios |
| **Confianza** | Comunicación 1 a 1, como hablar con alguien de confianza |

Crepes & Waffles es reconocido por su experiencia del usuario. Un cliente que pide por WhatsApp espera ser atendido con la misma calidez que en el restaurante. ALICIA hace exactamente eso.

## Arquitectura de Integración

### Flujo Operativo

```text
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Cliente    │────▶│   ALICIA    │────▶│   POS       │
│  WhatsApp   │     │   (IA)      │     │  Crepes     │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                    ┌──────────────────────────┼──────────────────────────┐
                    ▼                          ▼                          ▼
              ┌───────────┐             ┌───────────┐             ┌───────────┐
              │  Cocina   │             │ Domicilios│             │ Reportes  │
              │  (Comanda)│             │  (Despacho)│            │  (Métricas)│
              └───────────┘             └───────────┘             └───────────┘
```

### Requisitos de Integración

Para implementar ALICIA, Crepes & Waffles debe proporcionar acceso a:

| Sistema | Tipo de Integración | Datos Requeridos |
|---------|---------------------|------------------|
| POS | API REST/Webhook | Crear pedido, consultar menú, precios |
| Cocina | Via POS o API separada | Envío de comanda |
| Domicilios | API REST/Webhook | Crear orden, asignar repartidor, tracking |
| CRM (opcional) | API REST | Historial cliente, preferencias |

### Preguntas Técnicas para IT de Crepes & Waffles

1. ¿El POS actual expone API REST? ¿Qué versión/proveedor?
2. ¿El sistema de domicilios tiene endpoints para crear órdenes programáticamente?
3. ¿Existe documentación técnica de los endpoints disponibles?
4. ¿Hay un ambiente de staging/pruebas para integración?
5. ¿Cuál es el proceso de autenticación (OAuth, API Key, otro)?
