

# Plan: Expand Anti-Hallucination Guardrail to All Order Statuses

## Problem
The Layer 2 guardrail (line 4528) only intercepts hallucinated confirmations when `freshOrderStatus` is `"pending_confirmation"`. In the 573114018678 case, the AI never generated the `---PEDIDO_CONFIRMADO---` tag, so `order_status` stayed at `"none"`. The guardrail didn't fire, and the AI's hallucinated "Tu pedido está confirmado..." was sent directly to the customer without any real order being created.

## Root Cause
The AI can hallucinate confirmation language at ANY order status — not just `pending_confirmation`. The current guardrail is too narrow.

## Solution

**Single file: `supabase/functions/whatsapp-webhook/index.ts`**

### Change 1: Expand the guardrail scope (around line 4528)

Instead of only checking `pending_confirmation` / `pending_button_confirmation`, check ALL non-final statuses. The condition becomes:

- `parsed` is null (no order tag was generated by AI)
- AND the AI response contains confirmation language (`confirmado`, `en preparación`, `empezará a preparar`, `pedido confirmado`, `listo tu pedido`)
- AND `freshOrderStatus` is NOT in a final state (`confirmed`, `emailed`, `sent`)

When triggered:
- If `freshOrderStatus` is `pending_confirmation` or `pending_button_confirmation`: replace with "Responde *Sí* para confirmar"
- If `freshOrderStatus` is `none` or `active` (AI hallucinated an entire order+confirmation without tags): replace with a safe message like "¿Me confirmas los productos que deseas pedir? 😊" — since there's no real order to confirm

### Change 2: Additional check — verify DB order exists

Before sending any AI response containing confirmation language, also query `whatsapp_orders` to verify an actual order exists for this conversation. If no order exists in the DB AND the AI says "confirmed", it's always a hallucination regardless of status.

## Technical Detail

```text
Current guardrail (line 4528):
  IF status IN (pending_confirmation, pending_button_confirmation)
  AND resp matches confirmation language
  → replace resp

New guardrail:
  IF parsed === null (no order tag in AI response)
  AND resp matches confirmation language
  AND status NOT IN (confirmed, emailed, sent)
  → query whatsapp_orders for recent order in this conversation
  → IF no order exists in DB:
      replace resp with safe message asking customer to clarify their order
  → ELSE IF status is pending_confirmation:
      replace resp with "Responde *Sí* para confirmar"
```

## What stays untouched
- `saveOrder()`, `validateOrder()`, `buildOrderSummary()` — no changes
- Layer 1 payment-proof keywords — no changes
- The deterministic confirmation block (line 3591-3677) — no changes
- Reservation flow, cancel patterns — no changes
